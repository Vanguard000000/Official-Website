How It Works

  • MediaPipe Tasks Vision runs in the browser via WebAssembly.
  • All models are bundled locally — no CDN, no network requests after page load.
  • Camera access uses the browser's built-in getUserMedia API.
  • Overlay rendering is done on a 2D canvas, synchronized with the video feed.

Requirements

  • A modern browser: Chrome, Edge, Safari, or Opera.
  • A camera connected and available to the browser.
  • Run from localhost — opening files directly breaks module loading and camera permissions.
MediaPipe task models and WASM files are bundled in this folder so the demos run fully offline after the initial page load.
