Learning Journal

A daily log of concepts I learn, prompts I use, and problems I solve during my web development journey.

Week 4: Integration & Deploy

Day 28

Deploying to Netlify

🚀

I deploy the portfolio to Netlify via Git. I learn about continuous deployment, custom domains, and how to configure build settings for a static site. The automatic HTTPS and instant rollbacks are incredibly useful. I replace this placeholder with a screenshot of my deployment dashboard.

Day 27

Polishing Animations & Responsiveness

✨

I refine the scroll-triggered reveal animations, fix edge cases on mobile (nav wrapping, rocket bar visibility), and test across Chrome, Firefox, and Safari. The IntersectionObserver API makes scroll-based animations performant and easy to implement. I add photos of my responsive testing here.

Week 3: JavaScript Basics

Day 21

DOM Manipulation & Event Handling

⚡

I learn how to select elements, add event listeners, and modify the DOM dynamically. I build a simple theme toggle and a live clock before integrating these patterns into the portfolio's scroll effects and active nav highlighting.

Day 19

Arrow Functions, Promises & Async/Await

I take a deep dive into ES6+ syntax. Understanding promises and async/await is a breakthrough moment — writing asynchronous code that reads sequentially makes API calls and data fetching much more approachable.

Week 2: Styling & Layout

Day 14

CSS Grid vs Flexbox

📐

I build a simple dashboard layout to understand when to use Grid (2D layouts) vs Flexbox (1D layouts). Grid is perfect for the skills and projects sections, while Flexbox handles the nav and hero content naturally.

Day 12

Responsive Design & Media Queries

I start mobile-first but quickly realize that desktop-first with max-width breakpoints works better for this portfolio's visual design. Key breakpoints at 768px (tablet) and 480px (phone) with appropriate layout shifts.

Week 1: Foundations

Day 7

Semantic HTML & Accessibility

I learn the importance of proper heading hierarchy, landmark elements, and ARIA attributes. I run accessibility audits and fix contrast issues and missing labels.

Day 5

First Conversation with an LLM

🤖

My first prompting session: asking an AI to explain the difference between inline, block, and inline-block elements. The key insight is asking for explanations with analogies rather than just code snippets. This changes how I approach learning with AI.

Day 1

Setting Up the Environment

I install VS Code, learn basic Git commands, set up GitHub, and create the project folder structure. The first commit is just the HTML boilerplate and this README.
